# RAG 与 TicketSystem 简历项目经历设计

## 目标岗位

面向 Java 后端与 AI 应用开发岗位，Java 工程能力为主线，RAG 工程化能力作为差异化亮点，同时体现 React、Vue 前端联调和完整交付能力。

## 项目身份

两个项目均按个人独立开发与重构项目描述。文案只使用能够由仓库实现支撑、且候选人可以在面试中展开说明的内容，不包装为商业生产项目，不虚构团队规模、业务规模、QPS、准确率或性能提升比例。

## 排列与篇幅

1. RAG 项目放在前面，突出 AI 应用工程化。
2. TicketSystem 放在后面，证明 Java 微服务、高并发和数据一致性能力。
3. 每个项目包含项目名称、项目定位、核心技术和 5 至 6 条职责亮点。
4. 前端内容各保留一条，作为完整交付能力补充，不削弱后端主线。

## RAG 项目重点

- 企业知识库问答产品定位，而非简单的大模型接口 Demo。
- 知识入库链路：文件或 URL、对象存储、解析、分块、Embedding、关系库与向量库。
- 在线问答链路：会话记忆、问题改写、意图识别、多通道检索、Rerank、Prompt 组装和 SSE 输出。
- 模型路由、首包探测、故障切换与流式任务取消。
- MCP 外部工具调用、参数提取及知识库与动态业务数据融合。
- Redis 公平排队、全局并发控制和节点级 Trace。
- React 管理端、聊天状态管理和流式协议解析。

## TicketSystem 项目重点

- Spring Cloud Alibaba 微服务票务系统定位。
- Gateway、Nacos、OpenFeign 组成的服务治理与调用体系。
- Redis、Redisson、Kafka 支撑高并发异步下单、热点缓存和订单超时取消。
- 下单请求受理、消息消费、结果表回写及前端轮询形成异步闭环。
- ShardingSphere 对用户、节目、订单和支付数据进行分库分表。
- 订单创建、支付、取消和库存回补中的状态流转与一致性处理。
- Sentinel 流控、服务间接口校验及登录安全改造。
- Vue 3 购票、订单和账户页面联调。

## 表达规则

- 使用“设计并实现”“重构”“封装”“基于……解决”等主动表达。
- 每条先写问题或目标，再写技术方案和结果，避免只堆砌技术名词。
- 不写无法证实的量化结果；如后续有压测数据，再替换为真实指标。
- 技术栈只列核心组件，细节放到职责亮点中。

## 验收标准

- 招聘方能快速识别 Java 后端、微服务、高并发和 RAG 关键词。
- 每条内容均能对应到仓库模块或核心链路，能够继续追问。
- 两个项目定位互补，不重复堆叠相同技术。
- 文案可直接粘贴到中文简历，并为项目时间保留占位符。
